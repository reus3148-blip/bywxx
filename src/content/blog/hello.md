안녕하세요. 이 공간은 제가 **배우고 만든 것들을 천천히 기록하기 위해** 만든 블로그입니다.
무언가를 끝까지 이해했다는 신호는, 결국 다른 사람이 알아듣게 설명할 수 있는 것이라 믿어서요.

## 이 블로그는 어떻게 만들어졌나요

요란한 빌드 도구 없이도 빠르고 가벼운 정적 블로그를 만들 수 있는 [Astro](https://astro.build)로 만들었습니다.

- **프레임워크** — Astro (콘텐츠 컬렉션 + 마크다운)
- **글** — `src/content/blog/` 폴더의 `.md` 파일
- **배포** — Vercel 커스텀 도메인 자동 배포

> 도구가 단순할수록 글쓰기에 집중하기 좋다고 느꼈습니다.
> 멋진 기능보다 *오래 가는 글*이 더 중요하니까요.

## 글은 어떻게 쓰나요

`src/content/blog/` 폴더에 마크다운 파일을 하나 추가하면 됩니다.
파일 맨 위 frontmatter에 제목·설명·날짜를 적고, 그 아래에 본문을 씁니다.

```markdown
---
title: '내 새 글'
description: '한 줄로 설명하는 글의 요지'
pubDate: 'May 14 2026'
---

여기에 본문을 씁니다.
```

저장하고 `main` 브랜치에 푸시하면 Vercel이 자동으로 빌드해서 배포해줍니다.

### frontmatter 필드 정리

| 필드 | 필수 | 설명 |
| --- | :---: | --- |
| `title` | ✓ | 글 제목. 목록과 상세 페이지에 모두 표시됩니다. |
| `description` | ✓ | 한두 줄 요약. SEO와 미리보기에 사용됩니다. |
| `pubDate` | ✓ | 발행일. `'May 14 2026'` 같은 형식. |
| `updatedDate` |  | 수정일(선택). 적으면 상단에 "수정 …"으로 표기됩니다. |
| `heroImage` |  | 대표 이미지(선택). 글 상단에 크게 노출됩니다. |

## 앞으로 다룰 것들

지금 머리에 있는 주제들은 대략 이런 것들입니다.

1. **만들면서 배운 것들** — 작은 프로젝트의 시작부터 배포까지
2. **개념을 다시 정리한 메모** — 어렴풋이 알던 것을 글로 다듬기
3. **도구 사용기** — 새로 써본 도구의 첫인상과 한 달 뒤 인상

순서는 그때그때 마음 가는 대로 갑니다.
